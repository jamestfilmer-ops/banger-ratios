THE ALBUMS & ARTISTS QUESTION — READ THIS
You asked: "Why do I have to manually import artists and album covers? There are millions."

You don't have to. Here's how it actually works:

  1. User types any album or artist name in the search bar
  2. Your site calls the iTunes Search API (free, no key, 100M+ albums)
  3. iTunes returns album art, tracklist, release date, genre — everything
  4. User clicks an album → goes to /album/[id]
  5. That page auto-saves the album to your Supabase database on first visit
  6. From then on it appears in community lists, the leaderboard, everywhere

Your album/[id]/page.js already does ALL of this correctly.
The albums page just needed a search bar added — that is the entire fix.
You will never need to manually import a single album or artist cover.

-- Banger ratio recalculator (called after ratings are saved)
create or replace function public.recalculate_banger_ratio(p_album_id bigint)
returns void language plpgsql security definer as $$
declare
  track_record record;
  total_score  numeric;
  rating_count int;
  banger_count int := 0;
  album_ratings int := 0;
begin
  -- Update each track's avg_rating, total_ratings, is_banger
  for track_record in select id from public.tracks where album_id = p_album_id loop
    select count(*), coalesce(avg(score),0) into rating_count, total_score
    from public.ratings where track_id = track_record.id;
    update public.tracks set
      total_ratings = rating_count,
      avg_rating    = round(total_score::numeric, 1),
      is_banger     = (rating_count > 0 and total_score >= 5)
    where id = track_record.id;
    if rating_count > 0 and total_score >= 5 then banger_count := banger_count + 1; end if;
    album_ratings := album_ratings + rating_count;
  end loop;
  -- Update album banger_ratio
  update public.albums set
    total_ratings = album_ratings,
    banger_ratio  = case when (select count(*) from public.tracks where album_id = p_album_id) = 0 then 0
                    else round(banger_count::numeric / (select count(*) from public.tracks where album_id = p_album_id) * 100, 1) end
  where id = p_album_id;
end; $$;

import { useState, useEffect } from 'react'
import { useRouter } from 'next/navigation'
import { supabase } from '@/lib/supabase'

export default function AlbumsPage() {
  const router = useRouter()
  const [query, setQuery] = useState('')
  const [results, setResults] = useState([])   // iTunes search results
  const [saved, setSaved]     = useState([])   // albums in our DB
  const [searching, setSearching] = useState(false)
  const [activeTab, setActiveTab] = useState('search') // 'search' | 'community'

  // Load community-saved albums on mount
  useEffect(() => {
    supabase.from('albums').select('*').order('created_at', { ascending: false }).limit(60)
      .then(({ data }) => setSaved(data || []))
  }, [])

  // Search iTunes API — no key needed, completely free
  async function searchItunes() {
    if (!query.trim()) return
    setSearching(true)
    setResults([])
    setActiveTab('search')
    try {
      const res = await fetch(
        `https://itunes.apple.com/search?term=${encodeURIComponent(query)}&entity=album&limit=20`

TROUBLESHOOTING
🔴 Pink glow not visible after patcher
Cause: Browser cached the old CSS.

Fix: Hard refresh: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows). Also try:  rm -rf .next && npm run dev

🔴 Albums page search does nothing
Cause: Missing state update or fetch error — usually a network issue in dev.

Fix: Open browser DevTools → Console tab → look for a red error after clicking Search. Most common: CORS issue. Make sure you are fetching from itunes.apple.com not a custom URL.

🔴 Artists page shows "Loading discography..." forever
Cause: iTunes lookup by artistId failed or returned no albums.

Fix: Open DevTools → Network tab → look for the itunes.apple.com/lookup request. Check it returns results. If the artist has no albums in iTunes, the array will be empty.

🔴 Leaderboard is empty
Cause: No albums have been rated yet — the table is empty.

Fix: Go to /albums, search for an album, click it, sign in, rate some tracks, hit Submit. Then check /leaderboards.

🔴 Profile page shows "Loading..." forever
Cause: Not signed in, or profiles table missing.

Fix: Go to /auth first and sign in. If signed in and still loading: run the SQL schema from Phase 1 to make sure the profiles table exists.

🔴 Settings page redirects immediately
Cause: Not signed in.

Fix: Sign in at /auth first. Settings requires authentication.

🔴 "images.domains is deprecated" warning in terminal
Cause: Old next.config.mjs.

Fix: The patcher fixes this. After running the patcher the warning should be gone.

🔴 Build error: Module not found @supabase/supabase-js
Cause: Package not installed.

Fix: Run:  npm install @supabase/supabase-js  then try npm run build again.
